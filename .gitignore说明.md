## Git 命令参考

### 检查被忽略的文件

```bash
# 查看哪些文件会被忽略
git status --ignored

# 查看特定文件是否被忽略
git check-ignore -v .env
```

### 如果已经提交了敏感文件

```bash
# ⚠️ 从历史记录中删除（谨慎使用！）
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env" \
  --prune-empty --tag-name-filter cat -- --all

# 或使用 BFG Repo-Cleaner（推荐）
bfg --delete-files .env
git reflog expire --expire=now --all && git gc --prune=now --aggressive
```

### 清理已被跟踪但现在要忽略的文件

```bash
# 从 Git 跟踪中移除但保留本地文件
git rm --cached .env
git rm --cached -r data/milvus/

# 提交
git commit -m "Remove ignored files from tracking"
```

## 大文件处理建议

### 使用 Git LFS

```bash
# 安装 Git LFS
git lfs install

# 跟踪大文件
git lfs track "*.pth"
git lfs track "*.bin"
git lfs track "data/embeddings/*.npy"

# 查看被 LFS 跟踪的文件
git lfs ls-files
```

### 使用外部存储

```bash
# S3
aws s3 sync data/vector_store/ s3://your-bucket/vector_store/
aws s3 sync s3://your-bucket/vector_store/ data/vector_store/

# 阿里云 OSS
ossutil cp -r data/vector_store/ oss://your-bucket/vector_store/

# 下载脚本示例
cat > scripts/download_data.sh << 'EOF'
#!/bin/bash
echo "Downloading vector store data..."
aws s3 sync s3://your-bucket/vector_store/ data/vector_store/
echo "Done!"
EOF
chmod +x scripts/download_data.sh
```

### 3. Pre-commit Hook

创建 `.git/hooks/pre-commit`:

```bash
#!/bin/bash

# 检查是否误提交 .env
if git diff --cached --name-only | grep -q "^\.env$"; then
    echo "❌ 错误: 不能提交 .env 文件！"
    echo "   如果需要共享配置，请使用 .env.example"
    exit 1
fi

# 检查是否有 API Key
if git diff --cached | grep -iE "(api_key|apikey|secret|password).*=.*[a-zA-Z0-9]{20,}"; then
    echo "⚠️  警告: 检测到可能的 API Key 或密码"
    echo "   请确认不是敏感信息"
    read -p "   继续提交吗? (y/N) " -n 1 -r
    echo
    if [[ ! $REPLY =~ ^[Yy]$ ]]; then
        exit 1
    fi
fi

echo "✓ Pre-commit 检查通过"
```

```bash
chmod +x .git/hooks/pre-commit
```

## 常见问题

### Q1: 我已经提交了 .env，怎么办？

```bash
# 1. 立即撤销（如果还没 push）
git reset HEAD~ .env
git commit --amend

# 2. 如果已经 push，需要清理历史
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env" \
  --prune-empty -- --all

git push origin --force --all
```

⚠️ **注意**: 如果仓库已公开，API Key 已泄露，需要**立即重新生成新的 Key**！

### Q2: 向量数据很大，怎么共享？

**方案1**: Git LFS
```bash
git lfs track "data/vector_store/*.index"
git add .gitattributes
```

**方案2**: 云存储 + 下载脚本
```bash
# 上传
aws s3 sync data/vector_store/ s3://bucket/vector_store/

# 下载脚本
./scripts/download_data.sh
```

**方案3**: 重新生成
```bash
# 提供生成脚本
python scripts/embed_documents.py
```

### Q3: 如何检查哪些文件会被提交？

```bash
# 查看将要提交的文件
git status

# 详细查看（包括忽略的文件）
git status --ignored

# 模拟提交（不实际提交）
git add .
git status
git reset  # 取消 add
```
