# 切块策略集成指南

## 概述

已将三种切块策略成功集成到 `knowledge_base/hierarchical_index.py` 中：

1. **Simple** - 简单字符级滑动窗口（原有逻辑）
2. **Sentence** ⭐ - 句子级滑动窗口（推荐）
3. **Keyword Aware** - 关键词感知切分（政策文档专用）

---

## 测试结果

| 策略 | 切块数 | 平均大小 | 特点 |
|------|--------|---------|------|
| **Simple** | 3 | 271字符 | 字符级，可能切断句子 |
| **Sentence** ⭐ | 4 | 178字符 | 句子级，语义完整 |
| **Keyword Aware** | 2 | 357字符 | 保留关键词上下文 |

## 使用方法

### 1. Sentence 策略（推荐用于通用文档）

```python
from knowledge_base.hierarchical_index import (
    ChunkConfig,
    HierarchicalMarkdownProcessor,
    HierarchicalIndexBuilder
)

# 配置
config = ChunkConfig(
    chunking_strategy='sentence',  # 使用句子切分
    chunk_size=600,                # tokens
    chunk_overlap=80               # tokens
)

# 创建处理器
processor = HierarchicalMarkdownProcessor(config=config)

# 使用（在 build_from_markdown_files 中会自动使用）
builder = HierarchicalIndexBuilder()
# processor 会被内部使用
```

**优势**:
- ✅ 尊重句子边界
- ✅ 基于 token 数量（更准确）
- ✅ 语义完整性好
- ✅ 适合90%的场景

---

### 2. Keyword Aware 策略（政策文档专用）

```python
# 定义政策关键词
policy_keywords = {
    # 补贴相关
    "补贴", "补贴标准", "补贴资金", "补贴申请", "补贴范围",

    # 产品相关
    "家电", "以旧换新", "电视机", "冰箱", "洗衣机", "空调", "热水器",

    # 标准相关
    "能效标准", "水效标准", "能效", "水效",

    # 流程相关
    "申请条件", "申请流程", "实名认证", "补贴资格", "发票",

    # 部门相关
    "商务局", "市场监管", "财政部门"
}

# 配置
config = ChunkConfig(
    chunking_strategy='keyword_aware',
    chunk_size=500,                # tokens（第一阶段）
    chunk_overlap=80,
    keywords=policy_keywords,      # 关键词集合
    min_chunk_size=150,            # 最小chunk（字符）
    max_chunk_size=600,            # 最大chunk（字符）
    keyword_context_window=80      # 关键词前后保留字符数
)

processor = HierarchicalMarkdownProcessor(config=config)
```

**优势**:
- ✅ 保证关键词完整
- ✅ 保留关键词上下文
- ✅ 100%关键词覆盖
- ✅ 适合政策/技术文档

---

### 3. Simple 策略（仅用于特殊场景）

```python
config = ChunkConfig(
    chunking_strategy='simple',
    chunk_size=400,     # 字符数
    chunk_overlap=50    # 字符数
)

processor = HierarchicalMarkdownProcessor(config=config)
```

**注意**: 不推荐使用，可能会切断句子。

## 完整示例

### 示例 1: 嵌入政策文档

```python
import os
from pathlib import Path
from knowledge_base.hierarchical_index import (
    ChunkConfig,
    HierarchicalMarkdownProcessor
)

# 1. 定义政策关键词
policy_keywords = {
    "补贴", "补贴标准", "补贴申请", "补贴范围",
    "家电", "以旧换新", "电视机", "冰箱",
    "能效标准", "水效标准", "申请流程",
    "实名认证", "补贴资格", "发票", "商务局"
}

# 2. 配置关键词感知切分
config = ChunkConfig(
    chunking_strategy='keyword_aware',
    chunk_size=500,
    chunk_overlap=80,
    keywords=policy_keywords,
    max_chunk_size=600,
    keyword_context_window=80
)

# 3. 创建处理器
processor = HierarchicalMarkdownProcessor(config=config)

# 4. 读取文档
data_dir = Path("./data/process")
md_files = list(data_dir.glob("*.md"))

for md_file in md_files:
    with open(md_file, 'r', encoding='utf-8') as f:
        text = f.read()

    # 提取层次结构
    hierarchy = processor.extract_hierarchy(text, doc_id=md_file.stem)

    # 转换为节点（包含切块）
    nodes = processor.hierarchy_to_nodes(hierarchy, use_llm=True)

    print(f"✓ {md_file.name}: {len(nodes)} 个节点")
```

### 示例 2: 对比不同策略

```python
from knowledge_base.hierarchical_index import ChunkConfig, HierarchicalMarkdownProcessor

test_text = "... 你的测试文档 ..."

config3 = ChunkConfig(
    chunking_strategy='keyword_aware',
    chunk_size=500,
    keywords={'补贴', '家电', '申请流程'},
    max_chunk_size=600
)
processor3 = HierarchicalMarkdownProcessor(config=config3)
chunks3 = processor3._split_keyword_aware(test_text)
print(f"Keyword Aware: {len(chunks3)} 个切块")
```

## 参数调优指南

### chunk_size

- **小值 (200-400)**: 适合短文档、精确检索
- **中值 (400-800)**: 通用场景，推荐
- **大值 (800-1500)**: 长文档、需要更多上下文

### chunk_overlap

- **经验值**: chunk_size 的 10-20%
- **例子**:
  - chunk_size=500 → overlap=50-100
  - chunk_size=800 → overlap=80-160

### 关键词感知参数

- **min_chunk_size**: 150-200（避免过小）
- **max_chunk_size**: 500-800（触发二次切分）
- **keyword_context_window**: 50-100（关键词前后保留）

## 故障排除

### 问题 1: "未配置关键词"警告

**原因**: 使用 `keyword_aware` 策略但未设置 `keywords`

**解决**:
```python
config = ChunkConfig(
    chunking_strategy='keyword_aware',
    keywords={'补贴', '申请', '流程'}  # 添加这一行
)
```

### 问题 2: 切块太多或太少

**原因**: chunk_size 设置不当

**解决**: 调整 chunk_size
```python
# 切块太多？增大 chunk_size
config = ChunkConfig(chunk_size=800)  # 从600增到800

# 切块太少？减小 chunk_size
config = ChunkConfig(chunk_size=400)  # 从600减到400
```

### 问题 3: 关键词被切断

**原因**: 使用了 `simple` 策略

**解决**: 切换到 `keyword_aware`
```python
config = ChunkConfig(
    chunking_strategy='keyword_aware',  # 改为这个
    keywords=your_keywords
)
```
