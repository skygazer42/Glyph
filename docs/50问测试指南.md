# 50问综合测试指南

## 📋 测试概述

本测试套件包含 **50个精心设计的测试问题**,覆盖系统的所有主要路由:

| 路由 | 测试数量 | 说明 |
|------|---------|------|
| `faq_cache` | 2 | FAQ快速命中 |
| `dialogue` | 3 | 闲聊/问候/告别 |
| `clarify` | 3 | 模糊问题澄清 |
| `workflow` | 4 | 用户画像查询 |
| `graph` | 4 | LightRAG关系图谱 |
| `rule_engine` | 6 | 规则计算引擎 |
| `text2sql` | 5 | SQL生成 |
| `knowledge` | 17 | 向量知识库检索 |
| **总计** | **50** | 全覆盖测试 |

## 🚀 快速开始

### 方式1: 使用启动脚本 (推荐)

```bash
./run_50_tests.sh
```

然后选择测试模式:
- **选项1** - 快速测试 (并行执行, 约15-20分钟)
- **选项2** - 完整测试 (顺序执行, 详细报告, 约40-60分钟)
- **选项3** - 仅前10问 (快速调试, 约3-5分钟)

### 方式2: 直接运行Python脚本

**快速测试:**
```bash

```

**完整测试:**
```bash
python3 test_50_comprehensive.py
```

## 📝 测试用例说明

### FAQ Cache 测试 (#1-2)

测试系统的FAQ快速响应能力:
- ✅ 应直接命中 `resources/faq/qa_pairs.json`
- ⚡ 响应时间 < 0.5秒
- 🎯 置信度 > 0.9

### Dialogue 测试 (#3-5)

测试闲聊和社交对话:
- "你好,在吗?" → `greeting`
- "辛苦啦..." → `farewell`
- "你都能做什么?" → `chit_chat`

### Clarify 测试 (#6-8)

测试模糊问题的澄清能力:
- "这个政策怎么样?" → 应要求用户明确"哪个政策"
- "能不能说说那个东西?" → 应触发澄清流程

### Workflow 测试 (#9-12)

测试用户画像查询:
- 需要提供 `user_id` 参数
- 测试用户: 12345(张三), 67890(李四), 88888(王五)
- 应从 `USER_PROFILE_DB_PATH` 读取数据

### Graph 测试 (#19-22)

测试LightRAG知识图谱:
- 关注"关系"、"联动"、"协同"等关键词
- 返回实体间的关联和流程
- ⏱️ 预期耗时较长 (30-90秒)

### Rule Engine 测试 (#23-28)

测试规则计算引擎:
- 汽车补贴计算
- 保险补贴计算
- 家电补贴计算
- 应返回具体金额和计算过程

### Text2SQL 测试 (#29-33)

测试SQL生成能力:
- 需要提供 `connection_id` 参数
- 包含"SQL"、"查询"、"数据库"等关键词
- 应生成可执行的SQL语句

### Knowledge 测试 (#34-50)

测试向量知识库检索:
- 覆盖各类政策文档
- 测试日期、金额、条件等细节提取
- 检验LlamaIndex和Milvus的协同

## 🔧 调试和故障排除

### 问题1: 超时错误

如果遇到超时,检查:
```bash
# 检查.env配置
grep TIMEOUT .env

# 应该看到
PERFORMANCE__QUERY_TIMEOUT=90  # 至少90秒
```

前端超时已调整为120秒 (`web/src/api/request.js`)

### 问题2: Milvus连接失败

```bash
# 检查Milvus状态
docker-compose ps | grep milvus

# 测试连接
python3 -c "
from pymilvus import connections
connections.connect('default', host='localhost', port='19530')
print('✅ Milvus连接正常')
"
```

### 问题3: 路由不匹配

如果某些路由不匹配,检查:

1. **意图识别** - 查看日志中的intent
2. **关键词** - 某些路由依赖特定关键词
3. **参数** - text2sql需要connection_id, workflow需要user_id

### 问题4: 性能太慢

优化建议:
- 减少并发数: `QuickTestRunner(max_concurrent=2)`
- 跳过graph测试 (最慢)
- 调整LightRAG的top_k参数
