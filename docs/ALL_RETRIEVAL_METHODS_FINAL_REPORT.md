## 🔍 技术实现对比

### 方案1: 纯向量检索

**实现**: `knowledge_base/milvus.py`

**架构**:
```
查询文本
  ↓
Embedding (DashScope text-embedding-v3)
  ↓
Milvus 向量搜索 (COSINE相似度)
  ↓
Top-K 结果
```

**优点**:
- ✅ 语义理解能力强
- ✅ 实现简单，易于维护
- ✅ 精准率 100%
- ✅ 速度适中（~2秒）

**缺点**:
- ⚠️ 对关键词敏感度低
- ⚠️ 需要向量数据库

**适用场景**: 标准业务查询，追求精准度和实现简单

---

### 方案2: 向量 + Reranker

**实现**: `knowledge_base/milvus.py` + `knowledge_base/rerank.py`

**架构**:
```
查询文本
  ↓
向量检索 (初召回 top_k=10)
  ↓
Reranker (DashScope gte-rerank-v2)
  ↓
重新排序，精选 top_k=3
  ↓
最终结果
```

**优点**:
- ✅ 精准率 100%
- ✅ 二次精选，结果质量更高
- ✅ 适合复杂查询
- ✅ Reranker API 速度快

**缺点**:
- ⚠️ 需要两次API调用
- ⚠️ 总延迟最高（~3秒）
- ⚠️ 成本略高（Embedding + Rerank）

**适用场景**: 重要业务查询，追求最高质量

---

### 方案3: 混合检索（增强版） ⭐⭐⭐

**实现**: `knowledge_base/hybrid_retrieval_enhanced.py`

```

  ↓

**核心技术**:

1. **意图识别规则**:

    "汽车补贴": {
        "keywords": ["汽车", "新车", "购车"],
        "doc_type": "汽车消费政策",
        "boost_keywords": ["汽车", "新车", "补贴", "申请"],
        "boost_factor": 2.0
    },
    ...
}
```

2. **查询增强**:

3. **文档类型加权**:

## 🔧 LlamaIndex 方案失败说明

**原计划**: 使用 LlamaIndex + Milvus 稀疏索引实现 BM25+Vector 混合检索

**实际结果**: ❌ 失败

**失败原因**:
1. Milvus 稀疏索引配置错误 (`only IP is the supported metric type for sparse index`)
2. 代理配置冲突 (`Unknown scheme for proxy URL socks://...`)
3. LlamaIndex 与 DashScope embedding 集成不稳定

**详细分析**: 见 [LLAMAINDEX_HYBRID_ISSUES.md](./LLAMAINDEX_HYBRID_ISSUES.md)

**结论**: 自研方案已达到100%精准率，LlamaIndex方案不再必要。
