## 💡 使用方法

### 方式 1: 在 hierarchical_index.py 中使用

```python
from knowledge_base.hierarchical_index import (
    ChunkConfig,
    HierarchicalMarkdownProcessor
)

# 推荐：Sentence 切片
config = ChunkConfig(
    chunking_strategy='sentence',
    chunk_size=600,
    chunk_overlap=80
)

# 政策文档：Keyword Aware
policy_keywords = {
    "补贴", "家电", "申请流程", "能效标准"
}

config = ChunkConfig(
    chunking_strategy='keyword_aware',
    chunk_size=500,
    keywords=policy_keywords,
    max_chunk_size=600
)

processor = HierarchicalMarkdownProcessor(config=config)
```

### 方式 2: 直接使用 LlamaIndex

```python
from llama_index.core import Document
from llama_index.core.node_parser import (
    SentenceSplitter,
    SentenceWindowNodeParser
)

# Sentence 切片
splitter = SentenceSplitter(
    chunk_size=600,
    chunk_overlap=80
)

# SentenceWindow 切片（问答系统）
window_splitter = SentenceWindowNodeParser(
    window_size=3,
    window_metadata_key="window",
    original_text_metadata_key="original_text"
)

# 使用
doc = Document(text="你的文档内容...")
nodes = splitter.get_nodes_from_documents([doc])
```

## 🎓 四种策略详细说明

### 1. Token 切片

**原理**: 固定token数量切分

**示例**:
```
"其核心概念包括文档加载器、节点解析器、索引和查询引擎。"
                                        ↓ 在30 tokens处切断
Chunk 1: "其核心概念包括文档加载器、节点解析器、索"
Chunk 2: "引和查询引擎。"
```

**问题**: ❌ 切断了"索引"这个词

**结论**: 不推荐用于中文

---

### 2. Sentence 切片 ⭐ 推荐

**原理**: 尊重句子边界，基于token数量

**示例**:
```
Chunk 1: "LlamaIndex 是一个用于构建 LLM 应用程序的数据框架。
         它提供了一套工具，可以帮助开发者..."
         (167 tokens, 完整句子)

Chunk 2: "节点解析器随后将这些加载的文档分解成更小、更易于管理的单元，称为节点。
         这些节点通常是句子或段落..."
         (164 tokens, 完整句子)
```

**优势**:
- ✅ 语义完整
- ✅ 句子不被切断
- ✅ 基于token，更准确

**推荐配置**:
```python
SentenceSplitter(
    chunk_size=600,      # 通用场景
    chunk_overlap=80     # 13%重叠
)
```

---

### 3. Sentence Window 切片

**原理**: 核心句子 + 前后窗口上下文

**结构**:
```
         ┌─────────────┐
         │  前3个句子  │ ← 上下文
         ├─────────────┤
         │  核心句子   │ ← 用于检索
         ├─────────────┤
         │  后3个句子  │ ← 上下文
         └─────────────┘
```

**工作流程**:
1. 检索时：使用**核心句子**匹配
2. LLM生成时：使用**完整窗口上下文**

**配置**:
```python
SentenceWindowNodeParser(
    window_size=3,  # 前后各3句
    window_metadata_key="window",
    original_text_metadata_key="original_text"
)

# 需要配合 MetadataReplacementPostProcessor 使用
from llama_index.core.postprocessor import MetadataReplacementPostProcessor

query_engine = index.as_query_engine(
    node_postprocessors=[
        MetadataReplacementPostProcessor(target_metadata_key="window")
    ]
)
```

**适用场景**:
- ✅ 问答系统（需要前后文）
- ✅ 长文档理解
- ✅ 复杂推理任务

---

### 4. Keyword Aware 切片

**原理**: 两阶段切分 + 关键词保护

**流程**:
```
原始文档
    ↓
【第一阶段】Sentence 初步切分 (500 tokens)
    ↓
【第二阶段】检测关键词
    ↓
  包含关键词且过大？
    ├─ Yes → 在关键词边界二次切分
    └─ No  → 保留原chunk
    ↓
最终Chunks（保证关键词完整 + 上下文）
```

**示例**:
```
关键词: ["补贴标准", "申请流程", "能效标准"]

原文: "...对个人消费者购买2级及以上能效或水效标准的补贴产品，
      按最终销售价格的15%给予补贴..."

切分后:
Chunk: "...购买2级及以上【能效或水效标准】的【补贴】产品，
       按最终销售价格的15%给予【补贴】..."
       (保留了关键词前后80字符的上下文)
```

**配置**:
```python
ChunkConfig(
    chunking_strategy='keyword_aware',
    chunk_size=500,                 # 第一阶段大小
    keywords={'补贴', '申请流程'},   # 关键词集合
    max_chunk_size=600,             # 触发二次切分阈值
    keyword_context_window=80       # 关键词前后保留
)
```

## 🔄 与现有系统集成

### 修改 `scripts/embed_documents.py`

```python
#!/usr/bin/env python3
"""嵌入文档到向量数据库"""

from knowledge_base.hierarchical_index import (
    HierarchicalIndexBuilder,
    HierarchicalMarkdownProcessor,
    ChunkConfig
)

# === 选择切块策略 ===

# 方案1: Sentence 切片（推荐）
chunk_config = ChunkConfig(
    chunking_strategy='sentence',
    chunk_size=600,
    chunk_overlap=80
)

# 方案2: Keyword Aware（政策文档）
policy_keywords = {
    "补贴", "补贴标准", "补贴申请", "补贴范围",
    "家电", "以旧换新", "能效标准", "水效标准",
    "申请条件", "申请流程", "实名认证", "发票",
    "商务局", "市场监管", "财政部门"
}

chunk_config = ChunkConfig(
    chunking_strategy='keyword_aware',
    chunk_size=500,
    chunk_overlap=80,
    keywords=policy_keywords,
    max_chunk_size=600,
    keyword_context_window=80
)

# === 创建处理器 ===
processor = HierarchicalMarkdownProcessor(
    config=chunk_config,
    enable_images=True
)

# === 构建索引 ===
builder = HierarchicalIndexBuilder(storage_dir="./storage")

# 读取文档
from pathlib import Path
data_dir = Path("./data/process")
md_files = [str(f) for f in data_dir.glob("*.md")]

# 构建索引（会自动使用配置的切块策略）
result = builder.build_from_markdown_files(
    file_paths=md_files,
    use_llm=True,  # 是否生成Section摘要
    enable_images=True
)

print(f"✓ 索引构建完成")
print(f"  文档数: {result['doc_count']}")
print(f"  Section数: {result['section_count']}")
print(f"  Chunk数: {result['chunk_count']}")
```

## 🎯 推荐配置矩阵

### 按文档类型

| 文档类型 | 推荐策略 | 配置 |
|---------|---------|------|
| **通用文档** | Sentence | chunk_size=600 |
| **政策文档** | Keyword Aware | keywords + max_size=600 |
| **技术文档** | Sentence | chunk_size=800 |
| **新闻文章** | Sentence | chunk_size=400 |
| **长篇报告** | Sentence | chunk_size=1000 |
| **代码文件** | Token | chunk_size=500+ |

### 按应用场景

| 场景 | 推荐策略 | 原因 |
|------|---------|------|
| **简单检索** | Sentence | 性能好，质量高 |
| **问答系统** | SentenceWindow | 需要上下文 |
| **关键词搜索** | Keyword Aware | 保证关键词完整 |
| **语义搜索** | Sentence + Semantic | 主题识别 |

## 🔧 故障排除

### 问题1: "切块数太多"

**原因**: chunk_size 太小

**解决**:
```python
# 从 chunk_size=200 增加到 chunk_size=600
config = ChunkConfig(chunk_size=600)
```

### 问题2: "关键词被切断"

**原因**: 使用了 `simple` 策略

**解决**:
```python
# 改用 keyword_aware
config = ChunkConfig(
    chunking_strategy='keyword_aware',
    keywords={'补贴', '申请流程'}
)
```

### 问题3: "重叠太多，存储浪费"

**原因**: chunk_overlap 设置过大

**解决**:
```python
# 保持重叠在 10-20%
config = ChunkConfig(
    chunk_size=600,
    chunk_overlap=80  # 13%，合理
)
```
